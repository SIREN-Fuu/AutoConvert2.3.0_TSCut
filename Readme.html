AutoConvert ver 2.3.0
by rndomhack

○ このバッチファイルについて
  TSファイルを自動エンコードするためのバッチファイルないしJScriptです。

○ 主な機能
  ＊ Avisynth(32bit) + x264を使用した自動エンコードが行えます。
     また、標準入出力に対応したエンコーダもプリセットにより対応可能です。
  ＊ 番組名やサービス(チャンネル)名によってavsファイルを振り分けることができます。
  ＊ ファイルやEDCBのprogram.txt、rplsinfoから情報を取得し、エンコード後の名前を決めることが可能です。
     さらにSCRenameを使えば、しょぼいカレンダーから正確な名前を取得できます。
  ＊ ドラッグアンドドロップでエンコードできるAutoConvertUtil、
     GUIで確認しながらエンコード出来るAutoConvertGUI、単体で使用するAutoConvertがあります。
  ＊ Avisynthを使用しているため、新たにavsファイルを追加して
     Bob化したり逆テレシネ化したりフィルターかけたりするようなこともできます。

○ 導入
  設定ファイルは使用しないものについては空白のままにしておいてください。
  Avisynthプラグインは標準のプラグインフォルダではなくて、
  Avisynthフォルダにコピーするようにしてください。
  Avisynthやそのプラグインはすべて32bitで揃えてください。

  ＃ 必要なもの
    ＊ Avisynth             http://sourceforge.net/projects/avisynth2/
      動画編集
        インストーラーに従ってインストール

    ＊ BassAudio            http://behappy.codeplex.com/
      BASSを使用して音声を読み込む
      BeHappyのpluginsフォルダに入っている
        BassAudio.dll → Avisynth\
        bass.dll → Avisynth\
        bass_aac.dll → Avisynth\

    ＊ TDeint               http://bengal.missouri.edu/~kes25c/
      インターレース解除
        TDeint.dll → Avisynth\

    ＊ DGMPGDec             http://neuron2.net/dgmpgdec/dgmpgdec.html
      AAC分離 TSデコード
        DGDecode.dll → Avisynth\
        設定の AutoConvert -> パス -> DGIndex に DGIndex.exe へのパスを記述

    ＊ avs2pipemod          http://csbarn.blogspot.jp/
      AVSからエンコーダ等への橋渡し
      32bit版を使用してください。
        設定の AutoConvert -> パス -> avs2pipemod に avs2pipemod.exe へのパスを記述

    ＊ neroAacEnc           http://www.nero.com/jpn/technologies-aac-codec.html
      AACエンコーダ
        設定の AutoConvert -> パス -> neroAacEnc に neroAacEnc.exe へのパスを記述

    ＊ x264                 http://x264.nl/
      H.264エンコーダ
        設定の プリセット(x264, x264_high, x264_lowなど) -> エンコーダー に x264.exe へのパスを記述

    ＊ MP4Box               http://gpac.wp.mines-telecom.fr/
      H.264とAACを結合
        設定の AutoConvert -> パス -> MP4Box に MP4Box.exe へのパスを記述

○ 設定
  ＃ 設定ファイルについて
    以前は設定ファイルに直接書き込む必要がありましたが、
    現在はAutoConvertSettingsがあるためその必要はありません。
    AutoConvertSettingsで不明な項目がある場合は参考にしてください。
    
    設定ファイルはJSON形式で書かれています。
    JSONは \ がエスケープとなりますので記述する際は \\ としてください。
    設定ファイルの文法的チェックは、
    http://jsonlint.com/ や http://www.jslint.com/
    などで行うことができます。
    また、パスは実行ファイルへのパス(〜.exeまで)を記入してください。
    (正確には、cscript等ではJSONオブジェクトが使用できません。
    そのためevalで評価を行なっていますので、本来のJSONとは異なります)
    以下JSONを知らない方への簡単な説明。
    --------------------------------------------------
    {
        "項目1": {
            "設定1": "内容1",         ←文字列は"で囲む
            "設定2": "内容2",
            "設定2": "内容3"          ←最後の項目には , は不要
        },
        "項目2": {
            "設定1": true,            ←真偽値は"で囲まない
            "設定2": false,
            "設定3": 1,               ←数値は"で囲まない
            "設定4": 2,
            "設定5": 3,
            "設定6": {
                "小設定1": "内容1",   ←子設定も同様
                "小設定2": "内容2",
                "小設定3": "内容3"
            },
            "設定7": 4,
            "設定8": 5
        }
    }
    --------------------------------------------------

      %STH%             開始時間の時

  ＃ Service.txt と Program.txt の設定
    この項目はAutoConvertSettingsの詳細設定タブのサービス、番組と同様のものです。
    --------------------------------------------------
    [
        {
            "name": "機動戦士ガンダムAGE",
                //タイトルから検索する名前
            "program": "ガンダムAGE",
                //番組名
            "file": "ガンダムAGE 第$ACnumber$話",
                //ファイル名 [任意]
            "dir": "ガンダムAGE",
                //ディレクトリ名 [任意]
            "param": {
                "preset": "x264"
            },
            "level": {
                "preset": 1
            }
        },
        {
            "name": "機動戦士ガンダムSEED",
            "program": "ガンダムSEED",
            "param": {
                "avs": "720p",
                "preset": "x264_low"
            },
            "level": {
                "avs": 1,
                "preset": 1
            }
        },
        {
            "name": "HDリマスター",
            "program": "",
            "param": {
                "avs": "1080p",
                "preset": "x264"
            },
            "level": {
                "avs": 2,
                "preset": 1
            }
        }
    ]
    --------------------------------------------------
    Service.txt はサービス名から、Program.txt はタイトルから、
    "name" に当てはまるものを検索し、パラメータを決定できます。
    また、複数当てはまる場合はレベルの高いほうが優先され、
    レベルが同じ場合は上の方が優先されます。
    デフォルト設定のレベルは0ですのでレベルは1以上を指定してください。

    ＊ 上記タイトル例 :
        "機動戦士ガンダムSEED 37 PHASE-39「アスラン」"
                  … "機動戦士ガンダムSEED"に当てはまり、
                     番組名は"ガンダムSEED"、avsは"720p"、プリセットは"x264_low"となる。
        "機動戦士ガンダムAGE  第49話「長き旅の終わり」"
                  … "機動戦士ガンダムAGE"に当てはまり、
                     番組名は"ガンダムAGE"、avsには記述がないのでデフォルト設定、プリセットは"x264"となる。
                     また、ファイル名は"ガンダムAGE 第49話"、ディレクトリ名は"ガンダムAGE"となる。
        "機動戦士ガンダムSEED HDリマスターセレクション  PHASE-01「偽りの平和」" 
                  … "機動戦士ガンダムSEED"と"HDリマスター"に当てはまり、
                     番組名は上が優先され"ガンダムSEED"、avsはレベルが2の下が優先され"1080p"、
                     プリセットは同じレベル1のため、上が優先され"x264_low"となる。

                    "param": {
                        "avs": "1080p"

                        "avs": 1
                    }

            "target": [

                    "param": {

    基本的に Service.txt と Program.txt と似ているため構造は割愛します。

    ジャンルがアニメだった場合、AutoVfrになり、
    ジャンルがドラマだった場合、avsが"1080p"となります。
    また、話数が01だった場合、avsが"720p"となります。

  ＃ Presetフォルダの avsファイルについて
    ＊ サンプルのようなものですので環境に合わせて変更してみてください
    ＊ __opt__ はnv, t2a, cm, faw, alt, program, serviceのパラメータに置換されます
       AutoConvertUtilは acu.avs として出力し、AutoConvertに渡しています
       これによりある程度avs側で処理ができるようになっています
    ＊ __ac__ はAutoConvertフォルダへのパスに置換されます
    ＊ __trim__ はTrim情報に置換されます
    ＊ __def__ はItsのdefファイルに置換されます
    ＊ __tmc__ はタイムコード出力先に置換されます
       また、Service.txt と Program.txt によってパラメータを追加することができます
    ＊ AutoVfrに使用するavsは、__vfr__ は動画情報の出力先に置換されます

  ＃ Presetフォルダの txt(プリセット)ファイルについて
    この項目はAutoConvertSettingsの詳細設定タブのプリセットと同様のものです。
    --------------------------------------------------
    {
        "type": 0,                    //エンコーダーのタイプ(0: x264, 1: H.264出力 2: その他)
        "ext": "mp4",                 //拡張子
        "enc": "C:\\TV\\x264\\x264.exe",
            //エンコーダへのフルパス
        "opt": "<video> --demuxer y4m --crf 20 ……… --aq-strength 0.6 -o <output>"
            //エンコーダのオプション
    }
    --------------------------------------------------
    ＊ サンプルのようなものですので環境に合わせて変更してみてください
    ＊ オプション(opt) は <video>, <audio>, <output>が利用できます。
       それぞれ 入力(y4m映像), 入力(wav音声)(エンコーダのタイプがその他のみ使用可), 出力に対応しています
       (例: FFMpegの場合 → -i <video> -i <audio> ……… <output>)

  ＃ Presetフォルダの Logoフォルダについて
    ＊ 透過性ロゴフィルタで使用するロゴデータ(ldgファイル)を サービス名.lgd で保存してください
       また、logoGuilloで使用する サービス名.lgd.autoTune.paramや、
       comskipの サービス名.logo.txt、サービス名.iniもこのフォルダに入れてください。
       サービス名はService.txtで指定してください。

○ 使用方法
  ＃ AutoConvertUtil.bat (通常)
    TSファイルをドラッグアンドドロップしてください。
    
  ＃ AutoConvertGUI.bat (GUI)
    バッチファイルにドラッグアンドドロップするか、「追加」ボタンで
    TSファイルを追加してください。(リストにTSファイルを
    ドラッグアンドドロップしても追加できますが、恐らくVisualStudioが必要です)
    パラメータを調整後、開始ボタンで処理が始まります。
    
  ＃ AutoConvertSettings.bat (設定)
    起動して、設定を変更後、保存ボタンまたは名前を付けて保存ボタン(プリセットのみ)で
    設定を保存出来ます。

  ＃ AutoConvert.bat (引数を指定して実行)
    "AutoConvert.bat 引数" で呼び出してください。

    ＊ 引数
      -i input.ts   入力TSファイル
      -o output     出力ファイル名
      -a avs        avsファイル (フルパス)
      -p preset     プリセットファイル (フルパス)
      -cm           CMカットのみ
      -cut          TSカットのみ
      -tss          TsSplitter使用
      -nv           DGDecNV使用
      -t2a          ts2aac使用
      -cs           comskip使用
      -lg           logoGuillo使用
      -jls          join_logo_scp使用
      -c2a          Caption2Ass使用
      -apm          AvsPmod使用
      -av           AutoVfr使用
      -faw          FakeAacWav使用

○ ヒント
  ＊ CMカットではcomskipが出力した"ファイル名.ts.avs"に含まれるTrim情報を取得しています。
     事前にTrim情報を"ファイル名.ts.avs"に記述することで、CMカットを行うことができます。
     apmパラメータはこの方法でCMカットを行なっています。
     また、"ファイル名.ts.avs"にTrim情報がある場合、cmパラメータが指定されている場合を除き、
     comskip、logoGuillo、join_logo_scpはスキップされます。
     例: DGIndexでd2vファイルを作成し(tsを直接読み込むとズレが生じるため)、Aviutl読み込んでCMカットしたあと、
         Trimエクスポートプラグインでカット情報を取得、"ファイル名.ts.avs"に貼り付けて保存など。
  ＊ 逆テレシネやBob化をしたい場合は、avsファイルの Deint_(last) を
     IVTC_(last) (逆テレシネ)や Bob_(last)（Bob化）に変えることによって、かんたんに出来るようになってます。
     また、720p_ivtc, 720p_bob などがはじめから用意されているのでそれを指定するのも手だと思います。
     ただし、逆テレシネは TIVTC.dll を必要とします。
  ＊ サービス名は、ファイル名、ts.program.txtのサービス名、rplsinfoのサービス名からService.txtの
     nameに当てはまるserviceになります。サービス名はロゴデータを参照する際に必要になりますので、
     Service.txtに載ってない局は適宜追加し、ファイル名にサービス名が無いかつts.program.txtがない場合は、
     rplsinfoを導入することをおすすめします。
  ＊ AutoConvertUtilはDGIndexNV、ts2aac、comskip、logoGuillo、join_logo_scp、AutoVfr、FAWのいずれで失敗した場合、
     それを使用しないでリトライすることができます。
     設定の AutoConvert -> 設定 -> autoRetry をチェックしておくと、自動リトライできます。
  ＊ AutoConvertは各パートで失敗した場合、設定の AutoConvert -> 設定 -> failurepause が設定している場合、
     そのパートをリトライすることができます。avsファイルの調整時などに便利です。
